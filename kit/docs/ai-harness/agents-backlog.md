# Agents backlog

Agentes presentes na referência `multi-agents` **não adotados agora** (fora do foco
Java 25 / Claude+Codex). Ficam documentados como backlog — trazê-los só com
justificativa concreta no repo.

## Stacks não usadas neste repo
- `python-specialist` — só se entrar Python.
- `go-specialist` — só se entrar Go.
- `frontend-specialist` — só se houver frontend.
- `mobile-native-specialist` — só se houver mobile.
- `jakarta-ee-specialist` — só se migrar para Jakarta EE puro.
- `data-engineering-aws-architect` — só com pipelines de dados/Spark.

## Reviewers fora do foco atual
- `finops-reviewer` — quando custo de cloud for prioridade.
- `compliance-reviewer` — quando houver requisito regulatório formal.
- `incident-response-reviewer` — quando houver processo de incidente maduro.
- `ad-dba-reviewer` — quando o modelo de dados/DBA for crítico.
- `dependency-versions-reviewer` — útil; pode ser promovido quando houver muitas deps.

## Plataformas adiadas
- Copilot, Gemini, Devin — foco atual é **Claude Code + Codex**.

## Critério para promover do backlog
1. Existe necessidade real e recorrente no repo.
2. Há `.ai/rules/` ou scripts locais que o agente reforce.
3. Não duplica um agente já existente.
4. Cabe no princípio de "mínimo viável de agentes".
